# Backbone Extension Notes

本次扩展将风格迁移主流程从“固定 `VGG19`”调整为“可切换 `VGG19` / `ResNet50`”。

## 定位

- `VGG19`：论文复现基线，保持原有 Gatys 风格迁移路径不变。
- `ResNet50`：扩展对比实验，用于观察更换特征骨干后的表现差异。

## 当前实现

- GUI 新增“特征骨干”选择项，可直接切换 `VGG19（论文基线）` 与 `ResNet50（扩展实验）`。
- CLI 新增 `--backbone {vgg19,resnet50}` 参数。
- 运行结果的 `JSON` 元数据会记录 `backbone` 字段。
- `warmup` 现在会同时预热 `VGG19` 与 `ResNet50` 权重。

## 设计原则

- 默认值仍为 `vgg19`，保证原论文复现路径不受影响。
- `ResNet50` 使用独立的特征提取节点与层预设，不复用 VGG 的卷积层编号。
- `paper_mode`、`enhanced_mode`、`keep_color`、`mask` 等现有功能继续可用。

## 报告或演示中的推荐表述

- 可以将 `VGG19` 解释为“严格对齐原论文实现思路的复现基线”。
- 可以将 `ResNet50` 解释为“在相同优化框架下，对特征提取骨干进行替换后的扩展实验”。
- 不建议把 `ResNet50` 写成“论文原始设定”，更合适的说法是“改进尝试”或“对比实验”。

## 结果解读建议

- 两种骨干都能完成风格迁移，说明扩展方案在工程上是可行的。
- 两种结果会有明显视觉差异，这正是“更换特征空间”带来的实验价值。
- 是否“更好”不应只看单次主观观感，建议结合内容保持、纹理表现、颜色稳定性与运行时间综合比较。
