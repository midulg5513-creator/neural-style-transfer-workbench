# 神经风格迁移项目中文操作手册与配置指南

## 1. 文档目的

本文档用于说明本项目在 Windows + NVIDIA CUDA 环境下的安装、配置、运行与排错方法。

截至 `2026-04-13`，项目已经完成：

- 基础环境检查
- 核心风格迁移管线
- CLI 命令行运行入口
- 输出图片与 JSON 元数据保存

当前最稳定、已完成验证的使用方式是：

- 使用 `CLI` 运行
- 使用 `GPU` 执行
- 先进行 `warmup` 预热，再做正式演示

当前 `GUI` 入口文件 `app_gui.py` 仍处于骨架阶段，不建议作为正式演示主入口。

---

## 2. 项目功能概览

本项目是一个本地桌面方向的神经风格迁移课程项目，核心功能如下：

- 输入一张内容图 `content image`
- 输入一张风格图 `style image`
- 可选输入一张掩膜图 `mask image`
- 使用 VGG19 特征和 Gatys 风格迁移方法进行优化
- 支持 `style_strength` 风格强度调节
- 支持 `keep_color` 保色
- 支持 `mask` 局部风格化
- 输出结果图片
- 同时保存一份 JSON 元数据文件，记录本次运行参数与设备信息

## 3. 运行前提

### 3.1 硬件要求

- Windows 电脑
- NVIDIA 显卡
- 显卡驱动正常
- `nvidia-smi` 可以正常执行
- 建议显存不少于 `8 GB`

本机已验证设备：

- GPU：`NVIDIA GeForce RTX 4060 Laptop GPU`
- 驱动版本：`566.24`
- `nvidia-smi` 显示 CUDA 版本：`12.7`

### 3.2 软件要求

- PowerShell
- Python
- 本地可创建 `.venv` 虚拟环境
- 可以从官方 PyTorch 源安装 CUDA 版 `torch` 和 `torchvision`

重要说明：

- 本项目是 `GPU-only` 方案
- 不提供 CPU 回退
- 如果 `torch.cuda.is_available()` 为 `False`，则运行会直接失败

## 6. 推荐安装方式

### 6.1 进入项目根目录

在 PowerShell 中进入项目目录：

```powershell
cd c:\Users\33124\Desktop\计算机视觉期末
```

### 6.2 创建虚拟环境

```powershell
python -m venv .venv
```

### 6.3 激活虚拟环境

```powershell
.\.venv\Scripts\Activate.ps1
```

如果 PowerShell 阻止脚本执行，可临时执行：

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\.venv\Scripts\Activate.ps1
```

### 6.4 升级 pip

```powershell
python -m pip install --upgrade pip
```

### 6.5 先安装 CUDA 版 PyTorch

这一步非常关键。

不要直接假设 `pip install -r requirements.txt` 一定会装到 CUDA 版 `torch`。本项目实际调试过程中，默认安装曾经拉到 CPU 版轮子，因此推荐优先显式安装 CUDA 版：

```powershell
python -m pip install torch==2.7.1 torchvision==0.22.1 --index-url https://download.pytorch.org/whl/cu126
```

如果后续你确定自己的显卡环境适合其他官方 CUDA 轮子版本，也可以改为对应官方源，但演示前必须重新验证。

### 6.6 再安装其余依赖

推荐执行：

```powershell
python -m pip install PySide6>=6.8,<6.9 scikit-image>=0.24,<0.26 pytest>=8.3,<9
```

如需按仓库基线继续补齐：

```powershell
python -m pip install -r requirements.txt
```

说明：

- 如果你已经先装好了 CUDA 版 `torch` / `torchvision`，再执行 `requirements.txt` 一般不会影响主版本方向
- 但演示前仍建议再次验证 `torch.cuda.is_available()`

## 8. 演示前预热

### 8.1 为什么要 warmup

本项目依赖预训练 VGG19 权重。

如果第一次运行时现场联网下载权重，会带来以下风险：

- 网络不稳定
- 下载失败
- 演示时间不可控
- 离线环境无法运行

因此演示前必须先执行一次预热命令。

### 8.2 warmup 命令

```powershell
python cli.py warmup
```

成功后会输出类似信息：

- VGG19 权重已缓存
- 当前验证通过的设备是 `cuda`

### 8.3 权重缓存位置

在 Windows 下，常见缓存路径类似：

```text
C:\Users\<你的用户名>\.cache\torch\hub\checkpoints\
```

本项目当前 VGG19 权重文件通常为：

```text
vgg19-dcbb9e9d.pth
```

演示前建议确认这个文件已经存在。

- `--content`
  - 必填，内容图路径

  - 必填，风格图路径

  - 可选，最长边缩放尺寸

```powershell

  --style-strength 1.0 `
```

## 11. 参数配置说明

### 11.1 `--steps`

作用：

- 控制优化迭代步数

当前范围：

- 最小值：`50`
- 最大值：`1000`
- 默认值：`300`

建议：

- 课堂演示建议先用 `50` 或 `100`
- 如果时间允许，再用 `300`
- 步数越大，耗时越长

### 11.2 `--style-strength`

作用：

- 对风格损失进行倍率调节

当前范围：

- 最小值：`0.1`
- 最大值：`10.0`
- 默认值：`1.0`

建议：

- `0.5` 到 `1.5` 较稳
- 值太大容易导致风格过强、内容结构损失明显

### 11.3 `--image-size`

作用：

- 控制输入图片最长边缩放尺寸

当前要求：

- 必须大于 `0`
- 默认值：`768`

建议：

- 显存和时间紧张时可降到 `512`
- 需要更快演示时可以用 `256` 或 `384`
- 若图片过大，LBFGS 优化时间和显存占用都会明显上升

### 11.4 `--keep-color`

作用：

- 在风格化完成后进行 Lab 空间保色处理

建议场景：

- 希望保留原始内容图整体色调时使用

### 11.5 `--mask`

作用：

- 控制局部区域风格迁移

建议场景：

- 只想对人物背景、衣物、天空、局部物体进行风格化

## 14. 常见问题与解决办法

### 14.1 提示 PyTorch 未安装

典型报错方向：

- `PyTorch is not installed in the current environment`

解决办法：

- 先确认已激活 `.venv`
- 再重新安装依赖

### 14.2 提示 CUDA 不可用

典型报错方向：

- `A CUDA-capable NVIDIA GPU is required`

解决办法：

1. 先执行：

```powershell
nvidia-smi
```

2. 再执行：

```powershell
python -c "import torch; print(torch.__version__); print(torch.cuda.is_available())"
```

3. 如果 `torch.cuda.is_available()` 为 `False`，说明当前装到的可能是 CPU 版 PyTorch

建议重新安装 CUDA 轮子：

```powershell
python -m pip install --force-reinstall torch==2.7.1 torchvision==0.22.1 --index-url https://download.pytorch.org/whl/cu126
```

### 14.3 提示图片路径不存在

说明：

- `--content`
- `--style`
- `--mask`

这些路径写错了，或者文件不在当前位置。

解决办法：

- 优先使用绝对路径
- 或先执行 `dir` 确认文件位置

### 14.4 提示图片格式不支持

解决办法：

- 把图片转换为 `.png` 或 `.jpg`

### 14.5 提示 `num_steps` 不合法

当前合法范围是：

- `50` 到 `1000`

例如 `--steps 1` 会直接失败，这是正常校验结果。

### 14.6 第一次运行太慢

可能原因：

- 首次加载权重
- 首次初始化 CUDA
- 输入图片太大
- 步数太多

解决办法：

- 先执行 `warmup`
- 降低 `--image-size`
- 降低 `--steps`
