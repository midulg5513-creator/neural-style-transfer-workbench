# 神经风格迁移项目中文操作手册与配置指南

本文档面向公开仓库使用者，说明项目的安装、运行、输出与排错方式。

## 1. 项目概览

本项目提供一个桌面版神经风格迁移工具，核心能力包括：

- `Gatys + LBFGS` 风格迁移流程
- `VGG19` / `ResNet50` 特征骨干切换
- `style_strength` 风格强度调节
- `keep_color` 保色
- `mask` 局部风格迁移
- `enhanced_mode` 与 `paper_mode`
- 输出图像与同名 `JSON` 元数据记录

## 2. 运行前提

- Windows
- NVIDIA GPU
- 已安装 CUDA 版 PyTorch
- `torch.cuda.is_available()` 返回 `True`

当前仓库按 `GPU-only` 方案设计，不提供 CPU 回退。

## 3. 安装步骤

### 3.1 创建虚拟环境

```powershell
python -m venv .venv
```

### 3.2 安装依赖

```powershell
.\.venv\Scripts\python -m pip install --upgrade pip
.\.venv\Scripts\python -m pip install -r requirements.txt
```

如果你的环境需要其他 CUDA 版本，建议先按官方方式安装匹配的 `torch` / `torchvision`，再补装剩余依赖。

### 3.3 检查 CUDA

```powershell
.\.venv\Scripts\python -c "import torch; print(torch.__version__); print(torch.cuda.is_available()); print(torch.cuda.get_device_name(0) if torch.cuda.is_available() else 'NO CUDA')"
```

## 4. 首次运行前预热

建议先执行一次：

```powershell
.\.venv\Scripts\python cli.py warmup
```

这会缓存所需预训练权重。Windows 下常见缓存目录类似：

```text
C:\Users\<你的用户名>\.cache\torch\hub\checkpoints\
```

## 5. CLI 使用方法

### 5.1 基础示例

```powershell
.\.venv\Scripts\python cli.py run `
  --content .\path\to\content.png `
  --style .\path\to\style.png `
  --output .\outputs\demo-result.png `
  --steps 300 `
  --style-strength 1.0 `
  --image-size 768
```

### 5.2 局部风格迁移

```powershell
.\.venv\Scripts\python cli.py run `
  --content .\path\to\content.png `
  --style .\path\to\style.png `
  --mask .\path\to\mask.png `
  --output .\outputs\masked-result.png `
  --steps 300 `
  --style-strength 1.0 `
  --image-size 768
```

### 5.3 常用参数

- `--backbone {vgg19,resnet50}`：切换特征骨干
- `--keep-color`：保留原图色彩分布
- `--enhanced-mode`：启用更强的风格化参数组合
- `--paper-mode`：切换到更接近论文示例的参数组合

## 6. GUI 使用方法

启动命令：

```powershell
.\.venv\Scripts\python app_gui.py
```

基本流程：

1. 选择内容图和风格图。
2. 可选选择遮罩图。
3. 确认输出路径。
4. 设置步数、风格强度、图像尺寸和骨干网络。
5. 选择是否保色、是否启用强化模式。
6. 点击“开始风格迁移”。

## 7. 输出说明

每次成功运行后通常会生成：

- 输出图片，例如 `outputs\demo-result.png`
- 元数据文件，例如 `outputs\demo-result.json`

`JSON` 中会记录：

- 输入图与输出图路径
- 步数、风格强度、图像尺寸
- 骨干网络与模式参数
- Python、PyTorch、CUDA 与设备信息

## 8. 常见问题

### 8.1 CUDA 不可用

先检查：

```powershell
nvidia-smi
.\.venv\Scripts\python -c "import torch; print(torch.__version__); print(torch.cuda.is_available())"
```

如果返回 `False`，通常说明当前安装的是 CPU 版 PyTorch，或 CUDA 环境与轮子版本不匹配。

### 8.2 图片路径不存在

请确认 `--content`、`--style`、`--mask` 指向真实文件，必要时使用绝对路径。

### 8.3 输出格式不支持

当前支持：

- `.png`
- `.jpg`
- `.jpeg`
- `.bmp`
- `.webp`

### 8.4 第一次运行较慢

首次加载权重和初始化 CUDA 时耗时较长，属于正常现象。建议先执行 `warmup`。
