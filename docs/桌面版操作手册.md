# 桌面版操作手册

本文档面向日常使用，重点说明如何启动程序、如何在图形界面中完成一次风格迁移，以及输出图片大小是如何确定的。

## 1. 使用前准备

运行本项目之前，请先确认以下条件满足：

- 当前环境为 Windows
- 已安装 NVIDIA 显卡驱动
- 已安装 CUDA 版 PyTorch
- `torch.cuda.is_available()` 返回 `True`
- 已在项目根目录创建并安装 `.venv`

推荐先执行一次环境确认：

```powershell
.\.venv\Scripts\python -c "import torch; print(torch.__version__); print(torch.cuda.is_available()); print(torch.cuda.get_device_name(0) if torch.cuda.is_available() else 'NO CUDA')"
```

## 2. 首次运行前预热

第一次正式演示前，建议先预热模型权重：

```powershell
.\.venv\Scripts\python cli.py warmup
```

预热成功后，`VGG19` 权重通常会缓存到：

```text
C:\Users\<你的用户名>\.cache\torch\hub\checkpoints\vgg19-dcbb9e9d.pth
```

## 3. 启动图形界面

在项目根目录执行：

```powershell
.\.venv\Scripts\python app_gui.py
```

启动后界面会显示：

- 顶部工作台标题区域
- 左侧“设备状态”“素材与输出”“生成参数”“执行控制”面板
- 右侧内容图、风格图、输出结果预览区域
- 下方“输出记录”区域，用于显示结果摘要与 JSON 路径

## 4. 图形界面操作步骤

一次完整的 GUI 操作流程如下：

1. 在“内容图”处点击“选择”，加载一张内容图。
2. 在“风格图”处点击“选择”，加载一张风格图。
3. 如果需要局部风格迁移，在“遮罩图（可选）”处选择遮罩图。
4. 在“输出图”处确认保存路径，默认是 `outputs\result.png`。
5. 在“生成参数”中调整：
   - `优化步数`
   - `风格强度`
   - `原图保留度`
   - `图像尺寸`
   - `保留原图色彩`
   - `强化模式`
6. 如需更强风格化效果，可点击“一键应用强化参数”。
7. 点击“开始风格迁移”。
8. 等待进度条完成。
9. 在右侧查看生成结果预览，并在“输出记录”区域查看：
   - 输出图像路径
   - JSON 参数文件路径
   - 运行设备
   - 步数、风格强度、图像尺寸等信息

## 5. 推荐验证参数

如果只是想先快速确认链路是否正常，推荐先用一组较轻参数：

- 优化步数：`50`
- 风格强度：`1.0`
- 图像尺寸：`256`
- 保留原图色彩：可按需求开启
- 强化模式：建议先关闭，确认基础链路正常后再开启

这组参数适合作为冒烟测试。

### 5.1 强化模式说明

强化模式会自动启用更强的风格化参数组合，适合在你希望纹理更明显、笔触更强的时候使用。

当前强化模式包含：

- 更高的默认优化步数
- 更强的风格强度
- 更低的原图保留度
- 更偏向经典论文设置的初始化与池化策略

如果你只想快速确认链路是否正常，优先用普通参数做冒烟测试；如果你要追求更强的风格表现，再启用强化模式。

## 6. 输出图片大小说明

### 6.1 输出大小由什么决定

本项目不是直接输出原图像素尺寸，而是先按照 `image_size` 对输入图进行缩放，再输出相同尺寸的结果图。

规则如下：

- `image_size` 控制“最长边”的目标尺寸
- 另一条边会按原始宽高比等比例缩放
- 最终输出图与送入模型的内容图尺寸一致

### 6.2 如何预估自己的输出尺寸

如果原图是横图，且宽大于高：

- 最终宽度会接近 `image_size`
- 最终高度按比例缩小

例如：

- 原图：`1280 x 720`
- `image_size=768`
- 输出约为：`768 x 432`

如果原图是竖图，且高大于宽：

- 最终高度会接近 `image_size`
- 最终宽度按比例缩小

例如：

- 原图：`900 x 1200`
- `image_size=768`
- 输出约为：`576 x 768`

## 7. CLI 快速运行方式

如果你不通过 GUI，也可以直接用命令行运行：

```powershell
.\.venv\Scripts\python cli.py run `
  --content .\path\to\content.png `
  --style .\path\to\style.png `
  --mask .\path\to\mask.png `
  --output outputs\manual-demo.png `
  --steps 50 `
  --style-strength 1.0 `
  --image-size 256 `
  --keep-color
```

运行成功后会生成：

- `outputs\manual-demo.png`
- `outputs\manual-demo.json`

## 8. 常见问题

### 8.1 “开始生成”按钮不能点

通常表示当前 CUDA 不可用，请先检查：

- 是否装的是 CUDA 版 `torch`
- `torch.cuda.is_available()` 是否为 `True`

### 8.2 输出图片大小和原图不一样

这是正常现象。

因为程序会先按 `image_size` 对最长边做缩放，再把缩放后的内容图送入模型，最终输出结果也会保持这个尺寸。

### 8.3 结果图没有生成

请优先检查：

- 内容图路径是否存在
- 风格图路径是否存在
- 输出后缀是否为 `.png`、`.jpg`、`.jpeg`、`.bmp` 或 `.webp`
- 当前 GPU 环境是否可用

## 9. 最终检查

建议按这个顺序再确认一遍：

1. `nvidia-smi` 正常
2. `cli.py warmup` 正常
3. GUI 可以打开
4. 选择图片后内容图和风格图预览正常显示
5. 用 `50 / 1.0 / 256` 这组参数跑一次快速测试
6. 确认结果图和 JSON 文件都已写入 `outputs/`
